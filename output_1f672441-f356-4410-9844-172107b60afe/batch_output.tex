\centerline{\textbf{What is an image:}}
\vspace{0.5em}
\hrule

\begin{itemize}
    \item An image is just a picture made up of tiny dots called \textbf{pixels}.
    \item Each pixel has a value (like brightness or color).
    \item When all pixels are combined, they form the full picture you see.
\end{itemize}

\vspace{1em}

An image can be thought of as value a mathematical function $F(x,y)$ @$(11,7) = 128$.

\begin{figure}[h!]
    \centering
    \begin{tikzpicture}

    \end{tikzpicture}
    \caption{\textit{An image represented in a coordinate system with a point at $(x,y)$}}
\end{figure}

\begin{itemize}
    \item Here $x$ and $y$ = position of a point ($x$-axis, $y$-axis) on the image (pixel coordinates).
    \item $F(x,y)$ = the brightness (or intensity / gray level) at that point.
\end{itemize}

\vspace{1em}

\noindent \textbf{Gray level (intensity)}
\vspace{0.5em}

This tells how bright or dark that point is.
Example: In a black \& white photo $\rightarrow$ 0 = black; 255 = white, values in between = shades of gray.

\underline{Digital Image}

In the real world, x,y and intensity can have infinite values (continuous). That's an analog image (like film photography).

But computer can only handle finite, discrete values.
\begin{itemize}
    \item So we break the image into pixels (tiny squares).
    \item Each pixel has a number for intensity or color.
\end{itemize}
This converted form is called a digital image.

A digital image is a 2D signal
A digital image = a grid of pixels where each pixel has an intensity value (brightness or color).

\par\vspace{0.5em}\rule{\linewidth}{0.4pt}\par\nobreak\vspace{0.5em}
\centering\textbf{Where Does digital image processing Exist?}
\par\nobreak\vspace{0.5em}\rule{\linewidth}{0.4pt}\par\vspace{0.5em}

Digital IP sits between raw images data and intelligent understanding of images.
\begin{itemize}
    \item DIP starts with raw images (from camera, scanner, satellite, etc.).
    \item It focuses on improving or transforming images.
    \item But DIP doesn't "understand" what's inside the image, that's where image Analysis and computer Vision
\end{itemize}

\noindent\textbf{\underline{Boundaries of image processing}}
\bigskip

\noindent Where it Starts: $\rightarrow$ When you take a raw image and
want to improve it (remove noise, resize, sharpen \& enhance
contrast).

\noindent Where it stops $\rightarrow$ once you start understanding the objects
inside (like recognizing a car, face or text).
so DIP is mostly about the input = image, output =
image.
\bigskip

\noindent\textbf{\underline{Three Levels of Processing:}}
\bigskip

\begin{enumerate}
    \item \textbf{Low -level (Image $\rightarrow$ Image)}
    \begin{itemize}
        \item Works directly on pixels.
        \item Tasks:
        \begin{itemize}
            \item Noise Reduction (remove unwanted dots/grains).
            \item Resize or crop.
            \item Enhance brightness /contrast.
            \item Convert color $\rightarrow$ grayscale.
        \end{itemize}
    \end{itemize}
\end{enumerate}
\bigskip

\noindent\textbf{Example (car of number Plate):}
You take the original photo of the car.
It's noisy or dark $\rightarrow$ so you clean it (remove noise,
adjust brightness).
Then you apply gray scale filter (black \& white).
